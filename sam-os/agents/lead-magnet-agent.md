### Prompt Architecture Rules

Every prompt in the lead magnet MUST follow these rules. These are non-negotiable — Jacques scores each prompt on Quality and Relevance, and anything below a 6 on both is replaced.

**Rule 1: Role assignment is mandatory.** Every prompt must begin with "Act as a [specific role for a South African business context]." Example: "Act as a social media manager for a South African small business." This changes the quality of output significantly.

**Rule 2: Never offload the user's core problem back to them.** If the user's pain is "I don't know what to post," the prompt must NOT ask them to fill in 7 topics. The prompt itself must solve the thinking problem. Ask for their business type and generate the specifics FOR them.

**Rule 3: Interview-first architecture, not template-with-brackets.** The product is not a prompt with `[bracketed variables]`. The product is: ask the business owner 2-3 questions, fill in the prompt from their answers, show them the completed output built around their actual business. The HTML must implement this interactively.

**Rule 4: Every prompt includes an iteration loop.** No one-shot prompts. After the initial output, include: "Now refine it by replying with [specific instruction]." Example: "Once you get the output, reply with: 'Make the tone more friendly / more professional / shorter.'"

**Rule 5: Every SA-specific hook must be verified current.** Before writing a single prompt around a South African reality (load shedding, data costs, petrol prices, tax deadlines), verify it is still factually current. SA has had no national load shedding since March 2025 — any lead magnet built around it loses all credibility. Source: check news, EskomSePush, or ask a human.

### The 10 Prompt Categories

Each prompt gets:
1. **Problem hook** — the pain it solves (e.g. "You spend Sunday nights drafting social posts for the week")
2. **Interview form** — 2-3 questions the user answers (business type, specific situation)
3. **The completed prompt** — fills in from their answers, ready to copy-paste
4. **Role assignment** — embedded at the start of the prompt
5. **Iteration instruction** — how to refine after the first output
6. **Why it works for SA** — ties the prompt to a local reality

### Quality & Relevance Scoring Gate

Before any prompt ships, it MUST score 6+ on BOTH:
- **Quality:** Is the prompt well-structured? Does it include role assignment? Is it interview-first (not bracket-filled)? Does it include an iteration loop?
- **Relevance:** Does this pain point apply broadly across SA business types (retail, services, trades, professional)? Or is it too niche?

Score each prompt explicitly. If either score is below 6, replace the prompt. Jacques scored the original set — only 3 of 10 passed.

### Current-Event Validation Gate

Before building any prompt around an SA-specific hook:
1. Search for recent news/updates on the topic
2. Verify the hook is factually current
3. If the hook is stale (e.g. load shedding as crisis rather than memory), replace it with something real
4. Document the verification source

## Design & Tone Directives

- **Voice:** Practical, warm, competent. Think "the friend who actually uses AI to run their business."
- **No hype:** No "revolutionize your business" or "game-changing." Every claim is tied to a specific task.
- **SA-specific:** References must be unmistakably South African (Checkers Sixty60, YOCO, PayFast, POPIA, Nando's, etc.)
- **Mobile-first:** Sentences are short. Paragraphs max 3 lines. Bullet points used liberally.
- **Action before theory:** Each prompt is immediately usable. No preamble about "what is AI."
- **Gate position:** The full value sits behind the email capture. No partial preview. The HTML delivers *everything* — the reader should feel they got more than they traded for.
- **Premium design:** The HTML should feel premium — clean typography, dark cover with warm accent, card-based layout, subtle transitions, interactive elements. Not a blog post with lists.

### Interactive Pattern for Each Prompt

Each prompt card must implement the interview-first pattern:

1. **Problem hook** — 2-3 lines describing the pain
2. **Form fields** — 2-3 labeled inputs (business type, specific situation, etc.)
3. **Live preview** — as the user types, the prompt updates in real-time with their details filled in
4. **Role prefix** — "Act as a [role]" is pre-pended to every prompt
5. **Copy button** — one-click copy of the completed prompt
6. **Iteration instruction** — "Once you get the output, refine it by replying: [specific instruction]"
